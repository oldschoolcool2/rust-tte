# fixtures/ — the immutable contract

These Parquet files are **generated by the Oracle** (`../oracle/`) and are the
ground truth the Rust engine must reproduce. **Do not hand-edit them**, and do not
edit them to make a test pass — regenerate them from the pinned Oracle instead.

- **Format is Parquet, never CSV** — CSV silently coerces int/categorical/NA
  typing and round-trips floats, manufacturing false mismatches.
- Each cohort yields `input_<case>.parquet` plus `expected_<case>_itt.parquet`
  and/or `expected_<case>_pp.parquet`.
- `MANIFEST.json` records the Oracle provenance (package + R version, UTC
  timestamp) and a sha256 for every file. CI regenerates fixtures and fails on any
  sha256 drift, so the contract cannot silently change.

## Layout

| Subdir | Source | Tier |
|---|---|---|
| `scenarios/` | `oracle/20_scenarios.R` simulated cohorts | 1 — fidelity (bit-exact) |
| `edge/` | `oracle/30_edge_cases.R` immortal-time landmines | 1 — fidelity (bit-exact) |
| `harvested/` | upstream `tests/testthat` snapshots | 1 — fidelity (bit-exact) |
| `itt/` | ITT-specific cases | 1 — fidelity (bit-exact) |
| `pp/` | per-protocol cases | 1 — fidelity (bit-exact) |
| `weights/` | weight-application cases | exact join + ~1e-12 float |
| `golden/` | whole-pipeline coefficient tables (`oracle/50_*`) | 2 — integration (tolerance) |

The **structural columns** matched exactly are:
`id, trial_period, followup_time, assigned_treatment, treatment, outcome`
(plus a per-protocol censoring flag, once frozen from the pinned Oracle version).
