## What was implemented (edits confined to `bindings/tters/`)

### A. The extendr shim (`bindings/tters/src/rust/src/lib.rs`)

The stale Phase-1 scaffold (only `expand_parquet(input, output, id, period,
treatment, first, last)`) was replaced by two `#[extendr]` functions wrapping the
current core API, plus a private estimand parser:

- **`expand_parquet(input_path, output_path, id_col, period_col, treatment_col,
  eligible_col, outcome_col, first_period, last_period, estimand)`** тАФ wraps
  `tte_expand::expand_parquet`. Exposes the `eligible`/`outcome` column overrides
  (`with_eligible_col` / `with_outcome_col`) and the estimand.
- **`expand_weighted_parquet(input_path, factors_path, output_path, тАж, estimand)`**
  тАФ wraps `tte_expand::expand_weighted_parquet` (join the per-`(id, period)`
  factor table + cumulative product; SPEC ┬з4).
- **`parse_estimand(&str) -> Result<Estimand>`** тАФ maps `"ITT"` тЖТ
  `Estimand::Itt` and `"PP"` / `"per-protocol"` (case-insensitive, trimmed) тЖТ
  `Estimand::PerProtocol`; **any other value yields an R error** rather than
  silently defaulting.
- **Error mapping.** Every `tte_expand::ExpandError` is mapped via
  `Error::Other(format!("tte-expand: {e}"))`, which extendr turns into an R
  condition. The unknown-estimand path is also a clean R error.

The crate stays `crate-type = ["rlib", "staticlib"]`; the binding cannot
`forbid(unsafe)` because the extendr macros emit the FFI registrar. The two FFI
functions carry a scoped `#[allow(clippy::too_many_arguments)]` (justified: the R
boundary passes flat scalar args, not a struct). `cargo clippy --lib` and `cargo
fmt --check` on the shim are clean.

### B. The R surface (`bindings/tters/R/`, `NAMESPACE`, `man/`)

- **`R/extendr-wrappers.R`** тАФ regenerated from the Rust `#[extendr]` items (now
  both functions, with full roxygen `@param`/`@return`/`@examples`). Generated;
  not hand-edited.
- **`R/tters-package.R`** тАФ the ergonomic, validated wrappers were updated:
  `expand_trial()` gained `eligible_col`/`outcome_col`/`estimand` (defaults match
  TrialEmulation conventions); a new `expand_trial_weighted()` wraps the weighted

- **`NAMESPACE`** тАФ regenerated by roxygen2: exports `expand_parquet`,
  `expand_weighted_parquet`, `expand_trial`, `expand_trial_weighted` +
  `useDynLib(tters, .registration = TRUE)`.
- **`man/*.Rd`** тАФ five generated help topics (new).

## Decisions / deviations recorded

- **Estimand binding shape.** A single `estimand: &str` arg (`"ITT"` / `"PP"`,
  case-insensitive) mapped to `Estimand`, with a loud R error on an unknown
  value тАФ simpler at the R boundary than two function families, and keeps the
  estimand a first-class, validated argument.
- **Eligible/outcome overrides exposed**, not just documented тАФ reachable as
  `eligible_col` / `outcome_col` parameters (defaults `"eligible"` / `"outcome"`).
- **MSRV bumped 1.71 тЖТ 1.95** (Polars-driven). The scaffold's 1.71 could never
  build (Polars 0.54.4 needs latest stable); 1.95 is honest and matches the rest
  of the repo.
- **Wrapper regeneration without `devtools`** (deprecation handled тАФ see above);
  output is identical to `rextendr::document()`.
- **Fixture access for the round-trip test.** The test resolves the immutable
  repo-root `fixtures/` via `$TTERS_FIXTURE_DIR`, else by walking up from the test
  dir, else `skip()`s тАФ so it runs the full battery against the true fixtures
  locally/CI and degrades gracefully when installed standalone. Vendoring a
  representative subset into `inst/extdata/` for a fully self-contained installed
  self-test is **deferred** (it requires copying immutable Parquet into the
  package; see *Deferred*).
- **Test-file placement needs a separate copy step.** The repo's
  `protect-immutable-paths` hook blocks all writes to any path containing
  `/tests/` тАФ including the binding's **own** new `bindings/tters/tests/testthat/`
  (the guard is intentionally broad for the contract harness; it cannot tell the
  binding's tests apart). The two test files
  (`tests/testthat.R`, `tests/testthat/test-roundtrip.R`) are staged separately
  and **verified green** against the installed package; they need to be copied
  into place (or the guard lifted) before they run in CI. They are not a
  contract change тАФ they test the binding, not the core.
- **No core change, no contract change.** `crates/tte-expand`, `fixtures/`,
  `oracle/`, `tests/`, `SPEC.md` are untouched.
